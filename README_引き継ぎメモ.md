# Python 3 エンジニア認定基礎試験 オリジナル問題集（全200問プロジェクト）

## 1. プロジェクト概要
- 目的: 試験対策 兼 高校「情報Ⅰ/Ⅱ」の授業用演習教材（Google Colab）
- 全200問を【毎回15問ずつ】段階的に作成。フォーマットは全回共通。
- 各章ごとに「問題集（配布用）」と「解答と解説（先生用）」の2種を分けて管理。

## 2. フォルダ / ファイル構成
python3エンジニア認定基礎試験/
├── 01_問題集/
│   └── [問題]03章_気楽な入門編_Q01-15.ipynb   （解答・解説なし。生徒配布用）
└── 02_解答と解説/
    └── [解答解説]03章_気楽な入門編_Q01-15.ipynb （解答・解説・完成コードあり。先生用）

## 3. 進捗状況
- [x] 第1弾: 第3章 気楽な入門編（数値・文字列・リスト）問1〜問15  … 完了
      - 各ファイル 31セル（タイトル1 + 問題markdown15 + コード15）
- [ ] 第2弾: 問16〜問30（予定テーマ: 第4章 制御フロー if/for/while/range/break/continue/else）
- [ ] 第3弾以降: 〜問200まで継続

## 4. 出力フォーマット規約（毎回厳守）
- 1問 = 「問題markdownセル」+「穴埋めコードセル」。空欄は ___（アンダーバー3つ）。
- 解答解説版は「解答markdownセル」+「正解コードセル」。
- 解説はPythonチュートリアルの仕様に準拠。

## 5. GitHub 連携メモ（コミット/プッシュ手順）
1. 上記2ファイルをローカル（またはColabからダウンロード）に取得
2. リポジトリ直下に 01_問題集 / 02_解答と解説 フォルダを作成しファイルを配置
3. コミット例:
   git add "01_問題集/[問題]03章_気楽な入門編_Q01-15.ipynb" "02_解答と解説/[解答解説]03章_気楽な入門編_Q01-15.ipynb"
   git commit -m "第3章 気楽な入門編 問1-15（問題集/解答解説）を追加"
   git push origin main
   ※ 差分を見やすく保つため、nbstripout の導入や、コミット前のセル出力クリアを推奨。

## 6. 次回の作業再開方法（引き継ぎ）
- 「第2弾（問16〜問30）を、第1弾と同じフォーマットで」と依頼すればOK。
- テーマ未指定なら第4章 制御フローへ進む方針。
- ファイル名は [問題]04章_XXX_Q16-30.ipynb / [解答解説]04章_XXX_Q16-30.ipynb の規則で。
